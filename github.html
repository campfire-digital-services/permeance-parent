GitHub profile

This profile enables the github maven plugins for uploading artifacts and site information for hosting via github.

Configuration

To configure the plugins in this profile, add something like the following to your ~/.m2/settings.xml:

<server>
    <id>github</id>
    <username>martinlau</username>
    <password>sup3rs3cr3t</password>
</server>

The credentials contained in this setting will be used to publish the site and artifacts to the github repository specified by the projects scm.../scm element.

Included functionality

Artifact upload

The downloads-plugin is used to upload (yes, ironic) the artifacts resulting from the build to git hub for users to download.

It's probably worth noting that this plugin shouldn't be used to publish artifacts which are likely to be used as dependencies for other code. If this is your use case, you should be pushing to the Sonatype OSS repository.

Site publishing

The site-plugin is used to upload the generated maven site to github pages for hosting.

For an example, see the maven site for this project.

Customisation

There is no customisation beyond the initial setup of this profile. Either use it or don't!
